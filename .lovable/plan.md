## Авторизация по номеру телефона (без паролей и SMS)

Человек вводит номер → попадает в систему. Если номер новый — создаётся аккаунт. Если уже был — просто входит. Никаких кодов, паролей, подтверждений.

⚠️ Важно понимать: это **небезопасно**. Кто угодно, зная чужой номер, войдёт в его аккаунт. Подходит для прототипа/внутренней штуки, но не для продакшна с чувствительными данными. Если ок — поехали.

### Что меняется

**1. Edge-функция `phone-otp` → переписываем в `phone-login`**
Убираем всю логику с Twilio, кодами, таблицей `phone_otps`. Оставляем только:
- принимает `{ phone }`
- ищет пользователя в `auth.users` по телефону
- если нет — создаёт через `auth.admin.createUser({ phone, phone_confirm: true })`
- генерирует сессию (как сейчас: временный пароль + `signInWithPassword` + сброс пароля)
- возвращает `{ access_token, refresh_token }`

**2. `src/contexts/AuthContext.tsx`**
- Удаляем `sendPhoneOtp` и `verifyPhoneOtp`
- Добавляем `phoneLogin(phone)` — вызывает новую edge-функцию и ставит сессию через `setSession`

**3. `src/pages/Auth.tsx`**
- Убираем переключатель Email/Телефон, убираем весь email-флоу (или оставляем спрятанным — на ваш выбор; по умолчанию убираем целиком)
- Одна форма: поле "Номер телефона" + кнопка "Войти / Зарегистрироваться"
- Никакого экрана с кодом из SMS

**4. База / секреты**
- Таблица `phone_otps` больше не нужна — удаляем миграцией
- Секреты `TWILIO_API_KEY`, `TWILIO_PHONE_NUMBER` остаются неиспользуемыми (можно убрать руками в Connectors позже, не критично)
- `handle_new_user` триггер уже создаёт профиль из `display_name` или email — для phone-only пользователей `display_name` будет пустым, подставим сам номер телефона в `user_metadata.display_name` при создании

### Технические детали

- Email/пароль авторизация **полностью удаляется** из UI. Если позже захотите вернуть — скажите.
- Существующие email-пользователи в базе остаются, но войти через UI больше не смогут.
- Формат телефона валидируем на клиенте: должен начинаться с `+` и содержать 10–15 цифр.
- Edge-функция остаётся `verify_jwt = false` (публичный вход).
